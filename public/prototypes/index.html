Dashboard · Prototypes

Halfway-fidelity HTML mockups of every component, retained as a living record of decisions and discussions. Each prototype pairs a structural wireframe with numbered hotspots and an adjacent discussion panel.
How to read these. These are not final UI. The light theme + minimal styling is deliberate — it signals "we're still modelling". Numbered dots on each prototype are clickable discussion anchors. When a decision is made, the corresponding note gets tagged resolved and the whiteboard (docs/architecture/whiteboard.md) captures the outcome.

Domain entities — the pipeline core

vault-item.html The pipeline's centerpiece. Pass A (read-only) + Pass B (mutations, forms, junctions, deps) both landed. Archive/status/reassign all emit paired activity events. Attachments wired in thread composer. built
actors.html Who/what can act (marvin, ralph, boris, jimbo). Feature scaffolded at features/actors/. ActorSkill junction UI deferred. built
thread.html Discussion thread — service + reusable components at features/thread/. No routes (composed into VaultItem detail). Ready to be embedded. built
activity-feed.html Standalone view of an activity timeline (discriminated union of event types). Currently rendered inline in vault-item; a dedicated prototype would stress the shape. planned

Services — on top of the domain

routing.html Dispatcher simulator: pick a skill → see the decision trail. Capability matrix + routing summary. Sits on top of actors + actor-skills, not inside them. active
benchmarks.html Skills catalogue with per-skill champion model, quality bar, freshness, trend sparkline. Re-benchmark queue. Last-suite report card. active
kanban-actor-approval.html Kanban with per-card actor identity (Marvin / Scout / Forge / Muse / Sage) and a decomposition approval gate — agent-spawned subtasks land under the parent in grooming as proposed, with bulk approve/reject actions before they escape to Ready. active

Mobile UX — responsive redesign

mobile-kanban.html Mobile kanban (375px): sticky top nav replaces left sidenav; horizontal chip strip switches columns; new intake_rejected card variant surfaces last-activity excerpt + open-question count inline — no click required. Two states: intake_rejected column (centerpiece) + ungroomed. active
mobile-detail-tabs.html Mobile detail modal as full-screen takeover with 4-tab nav (Overview / Body / Activity / Discussion). Tab badges surface urgency (open questions, event count) without switching. Three screens: Overview tab, Activity tab, Discussion tab with compose. active
mobile-detail-stacked.html Alternative mobile detail layout: accordion sections (Status & Meta, Body, Activity, Discussion) on one scrollable page. Section headers carry badge counts. Includes a comparison table weighing tabs vs stacked. Two states: default (top sections expanded) + all-sections expanded. active
